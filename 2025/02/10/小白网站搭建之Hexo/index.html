小白网站搭建之Hexo

初步搭建环境

准备工具

首先需要下载 Node.js 和git 两个工具

注意：尽量不要安装到文件名有中文的，安装时一直点确定即可

接下来检查是否安装成功

打开cmd输入
npm -v 然后node -v 然后git -v

搭建github仓库

成功下载工具后，进入GitHub开始搭建GitHub储存静态页面的仓库

  • 首先注册/登录
  • 点击Create a new repository进入新仓库页面
  • 仓库名输入
  • 勾选Public和Add a README file
  • 最后点击create创建仓库

生成SSH Keys

进入任意文件夹，右击空白处点击Git bash here

  • 输入ssh-keygen -t rsa -C xxxx@xx.com
  • 然后敲4下回车

进入C:\User\用户名

  • 找到.ssh文件夹进入
  • 用记事本打开里面的id_rsa.pub文件，全选复制里面的内容

打开GitHub

进入用户设置，找到SSH keys

  • 新建SSH keys，名称随意，在下面粘贴刚才复制的内容
  • 然后点击创建

测试是否成功

在git bash中输入
ssh -T git@github.com

  • 回车，然后输入yes，出现successful就表示成功了

到这里前面的准备工作就结束了，下面开始进行博客的搭建

开始搭建hexo环境

首先在你想要存放博客文件的位置创建一个文件夹（名字任意）例如：blog或者boke文件夹

然后进入该文件夹打开Git bash here

下载hexo

在Git bash here里面输入
npm install hexo-cli -g

生成本地环境

下载完成之后再输入
hexo init
或者
npx hexo init

然后输入
hexo install

接下来一次输入
hexo g
hexo s

成功之后可以复制生成的链接进入游览器看到生成的本地服务器【本地环境】
Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.

注意：如果显示 ERROR Deployer not found: git，那就要用git bash输入npm install hexo-deployer-git --save

之后要用本地环境的，都用这个方法，然后ctrl+c关闭本地服务器（关闭本地服务器之后的localhost就出现被拒绝访问的页面了）

根目录中进入之前的创建的博客文件夹，用记事本或者vscode打开_config.yml文件

设置github自动部署

根目录中打开Git bash here

安装自动部署发布工具
npm install hexo-deployer-git --save

然后依次输入
hexo g
hexo d

如果是第一次使用git的话会需要配置

配置完后再使用hexo d上传

在跳出的github登录窗口进行登录操作

这样就成功的把本地内容上传到GitHub了

上传成功以后，去自己的github检查下是否有上传到代码内容，有就算搭建好了

博客网址是之前设的仓库名：==https://用户名.github.io==

更换主题

  • 在你的 Hexo 根目錄下打开git bash，输入

git clone -b master https://gitee.com/immyw/hexo-theme-butterfly.git themes/butterfly

  • 修改 Hexo 根目錄下的 _config.yml，把主題theme：改為 butterfly

為了減少升級主題後帶來的不便，請使用以下方法（**建議）：

在 hexo 的根目錄創建一個文件 _config.butterfly.yml，並把主題目錄的 _config.yml 內容複製到 _config.butterfly.yml 去。

注意:

複製的是主題的 _config.yml ，而不是 hexo 的 _config.yml

不要把主題目錄的 _config.yml 刪掉

以後只需要在 _config.butterfly.yml 進行配置就行。如果使用了 _config.butterfly.yml， 配置主題的 _config.yml 將不會有效果。

新增内容

方法1：

  • 根目录打开git bash
  • 新增发布文章输入 hexo new post xxx
  • 新增页面输入 hexo new page xxx

方法2：

  • 直接把md文件拉到source/post中

注意文章开头一样是通过输入yml语法来进行分类

Vercel代码托管【建议做】

GitHub 本身只是一个代码托管平台，它不会自动部署你的代码到生产环境
Vercel 是一个专注于前端开发和部署的平台，主要用于托管和部署静态网站、JAMstack 应用和 Serverless 函数。

vercal主要功能：

  • 部署：自动从 Git 仓库（如 GitHub、GitLab、Bitbucket）拉取代码并部署。
  • 持续集成/持续部署 (CI/CD)：每次代码推送后自动构建和部署。
  • Serverless 函数：支持无服务器函数，方便构建 API 和后端逻辑。
  • 性能优化：自动优化静态资源加载速度，支持 CDN 分发。
  • 预览环境：每次 Pull Request 都会生成一个预览 URL，方便团队审查。

使用步骤

  • 注册用户，关联自己的github账号
  • 登录后，点击 “New Project” 按钮。
  • 选择你要部署的 Git 仓库（例如 GitHub 上的某个仓库）。
    • 如果是第一次使用，Vercel 会请求访问你的 Git 仓库权限，授权即可。
  • 选择仓库后，点击 “Import”
  • 然后填写你的网址，如果有域名就填上域名，点击“deploy”
